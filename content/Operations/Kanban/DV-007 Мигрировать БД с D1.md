## Цель
Освободиться от привязки к D1 Bindings. Получить локальную БД, которую можно бэкапить и переносить.

## Definition of Done
- [ ] Принято решение: SQLite-файл (проще) или Postgres (масштабируемее). По умолчанию — SQLite
- [ ] Экспортирован дамп текущего D1 (`wrangler d1 export`)
- [ ] Дамп импортирован в новый инстанс
- [ ] В коде заменён клиент D1 на драйвер для выбранной БД (better-sqlite3 / postgres-js)
- [ ] Все миграции 0001-NNNN прогоняются на новой БД без ошибок
- [ ] Проверено локально: API возвращает те же данные, что и в продакшене на CF

## Зависимости
- Блокирует: [[DV-008 Деплой dv-hub на VPS (Node adapter + PM2)]]

## Заметки
SQLite подходит для текущего объёма. Postgres имеет смысл, если в фазе 2 будут мульти-инстансы или сложные индексы. На MVP — не over-engineer.
