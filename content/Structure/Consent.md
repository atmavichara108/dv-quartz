# Consent — принятие решений

## Что это
Consent — основной способ принятия решений в ДВ. Это не голосование большинством и не единогласие.
Решение принимается, когда **ни у кого нет обоснованного возражения**.

## Как это работает
1. Кто-то выносит **предложение** (текст, документ, идею)
2. Каждый участник высказывается — по кругу или в чате
3. Три варианта реакции:
    - **«Согласен»** — возражений нет, предложение работает
    - **«Предлагаю изменение»** — с конкретной формулировкой и аргументом
    - **«Возражаю»** — с объяснением почему предложение навредит общей цели
4. Если есть предложения по изменению — интегрируем и проводим consent заново
5. Если есть возражение — обсуждаем, ищем решение которое снимает возражение
6. Когда возражений нет — решение принято

## Что является возражением
Возражение должно быть **обоснованным** — связанным с общей целью и принципами ДВ. Фасилитатор помогает определить, является ли высказывание возражением.

**Это возражение:** «Если мы примем это, мы не сможем масштабироваться, потому что новые ячейки не поймут наши правила» — связано с целью децентрализации.
**Это НЕ возражение:** «Мне не нравится формулировка» — без объяснения как это вредит цели. «Я бы сделал по-другому» — личное предпочтение без аргумента.

## Почему не голосование
Голосование большинством означает что меньшинство проиграло. В горизонтальной системе это создаёт расколы. Consent означает что решение **достаточно хорошее для всех прямо сейчас** — не идеальное, но рабочее. Любое решение можно пересмотреть позже если появятся новые аргументы.

## Зачем нужен фундамент (Основа ДВ)
Consent работает только когда есть общая база — согласованные миссия, принципы и цели. Без этого невозможно определить, является ли возражение обоснованным, потому что нет критерия «навредит ли это нашей общей цели». Документ «Основа ДВ» — этот фундамент. Он сам принимается через consent первым.

## Где применяется
Consent используется для принятия любых значимых решений: утверждение документов, назначение ролей, приём новых участников, выбор тем, изменение процессов. Мелкие операционные решения (когда созвон, какой формат записи) можно принимать проще — тот кто несёт роль, решает в рамках своих полномочий.
